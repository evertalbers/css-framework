Intro & Compatability

This framework is based around a number of components that can each be used by themselves or as group. These components are:

  • Grid
  • Aligners
  • General Styling
  • Activestates (tabs)
  • Menus

In addition to these we usually use normalize.css as our reset and wallop or slick as a carousel.

Compatability

The components are compatible with modern browsers (IE11+). It is tested on:

  • IE11 (IE10 as long as you use percentages for horizontal paddings/margins for the grid)
  • Edge 12, 13
  • Safari 8, 9
  • Chrome
  • Firefox

Grid

The base grid is constructed with a container with the class flex, which then contains columns with classes one through twelve, or the additional class fifth

There are a few aditional classes in the grid, these are:

margin
Give all columns a margin of 1em
padding
Give all columns a padding of 1em
col
Switch the grid to vertical layout, see the Aligned equal columns example.

And these can be used on the columns themselves:

no-grow
The element will not grow to fill available space
no-shrink
the element will not reliquish space to allow other elements to grow

All flex and aligner related classes have three breakpoints which are used with prefixes:

.m-
Breakpoint at 1100px, example: .m-six
.s-
Breakpoint at 800px, example: .s-six
.xs-
Breakpoint at 500px, example: .xs-six

Example:

three m-four s-six xs-twelve no-grow
three m-four s-six xs-twelve no-grow
three m-four s-six xs-twelve no-grow
three m-four s-six xs-twelve no-grow

Since the aligners have the same breakpoints as the grid you can easily move the content at different screensizes. This text will start on the bottom right, and move to center and then to top left as window size changes. The classes for this are align bottom right m-center s-top s-left

General styling

All colors and fontsizes are controlled in a file called custom.css. I recommend that you use a tool such as gridlover to generate fontsizes for your site.

Menu

The grid comes with a basic menu, that is either fixed by adding the class fixed on the body element, or static (default).

The standard HTML for a header with menu looks like this: 

		<header class="flex padding">
			<div class="four m-six logo align center left">
				<a href="/">
						Home
					</a>
			</div>
			<ul class="menu eight m-six align center space-around">
				<li>
					<a href="#one">Item one</a></li>
				<li class="dropdown">
					<a href="#two">Item two</a>
					<ul>
						<li>
							<a href="#">Sub 1</a></li>
						<li>
							<a href="#">Sub 2</a></li>
						<li>
							<a href="#">Sub 3</a></li>
						<li>
							<a href="#">Sub 4</a></li>
					</ul>
				</li>
				<li>
					<a href="#three">Item three</a></li>
			</ul>
			<div class="menu-icon-wrapper m-align m-center m-right">
				<div id="menu-icon">
					<span class="first"></span>
					<span class="second"></span>
					<span class="third"></span>
				</div>
			</div>
		</header>

The JS for opening the menu on mobile/tablet is just four lines (no dependencies):

		document.querySelector("#menu-icon").addEventListener("click", function () {
			document.querySelector("ul.menu").classList.toggle("show");
			document.querySelector("#menu-icon").classList.toggle("active");
		});
